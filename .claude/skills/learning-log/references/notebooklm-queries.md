# NotebookLM クエリテンプレート

Step 3-3 で `notebook_query` ツールに渡すクエリのテンプレート。
3つのクエリを順番に実行する。各クエリ実行間に1秒以上のスリープを入れること。

---

## クエリ1: 全体要約

```
この内容を初学者にもわかるように、主要なポイントを5〜7個の箇条書きで要約してください。
専門用語には簡単な解説を付けてください。
日本語で回答してください。
```

**用途**: レポートの「学んだこと > 主要ポイント」セクションの素材になる。

---

## クエリ2: 構造化

```
この内容の構造を以下の観点で整理してください。日本語で回答してください:

- 前提知識（この内容を理解するために必要な知識）
- コアコンセプト（最も重要な概念や主張）
- 具体例やケーススタディ
- 実践的なアクションアイテム
```

**用途**: レポートの「前提知識」「詳細」セクションの素材になる。

---

## クエリ3: PROFILE連動（応用分析）

```
以下のプロフィールを持つ人にとって、この内容はどのように活かせますか？
具体的な応用シーンを3〜5個挙げてください。日本語で回答してください。

プロフィール:
{PROFILE.mdの関連部分を動的に挿入}
```

**用途**: レポートの「実務への応用」セクションの素材になる。

### PROFILE.md 挿入ルール

- `{PROFILE.mdの関連部分}` には、Step 1 で読み込んだ PROFILE.md から以下を抽出して挿入する:
  - 職業・メインの分野
  - 学習テーマに関連する興味領域
  - 「深掘りしたい分野」「まだ詳しくない分野」の該当箇所
- PROFILE.md 全体を挿入するのではなく、関連する部分のみを抜粋する（トークン節約）

---

## クエリ実行時の注意

- 各クエリの実行間に**1秒以上のスリープ**を入れる
- 応答が空または不十分な場合、プロンプトを具体化して**最大2回まで**再実行する
- 全てのクエリが失敗した場合は、WebFetch + Claude自身の分析にフォールバックする
