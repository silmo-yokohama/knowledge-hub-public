# 企画書テンプレート: new-idea

以下の形式で `02.Ideas/YYYY-MM-DD_{title}/overview.md` に出力する。

ファイル名ルール:
- `{title}` はアイデアのタイトルを短く表したもの（日本語可、英語可）
- ファイル名に使えない文字（`/`, `\`, `:`, `*`, `?`, `"`, `<`, `>`, `|`）は全角に変換する
- スペースはハイフン `-` に置換する

---

## テンプレート

```markdown
# {アイデアタイトル}

> カテゴリ: {技術系 / 事業 / 趣味・プライベート / その他}
> 作成日: {YYYY-MM-DD}
> ステータス: アイデア段階

---

## 概要

{アイデアの概要を2〜3文で簡潔に説明}

---

### What（何を）

{何を作るのか・何を実現するのか}

### Why（なぜ）

{誰が使うのか・ターゲットユーザー}

{いつまでに・どのタイミングで}

{どのプラットフォーム・どの環境で}

### How（どうやって）

{どのような技術・手段で実現するか}

---

## 実現可能性

{判断理由と想定コストの内訳}

### 差別化: {高い / 中程度 / 低い / なし}

{判断理由}

{判断理由}

```

## テンプレートの補足ルール

- 全てのセクションを埋める必要はない。情報が不足している場合は「未定」と記載する
- 技術スタックは PROFILE.md のスキルセットを考慮して提案する
- リスクは最低1つは挙げる（リスクゼロのアイデアは存在しない）
- MVP 定義はできるだけ小さく絞る（1〜2日で着手できる規模が理想）
- ネクストアクションは具体的かつ着手可能な粒度で記載する
